トランジションプロパティ基本のき

CSS - Transition 基本的な使用方法について

トランジションとは、要素の1つの状態から別の状態への変化を滑らかにするために使用されるCSSプロパティです。
アニメーションで使用させる場合は、一般的に、マウスを要素に重ねる（ホバーする）などのユーザーのアクションに応じて、要素の特定の属性（例：色、サイズ、位置など）が変化する場合、特にボタンなどに利用される事が多いです。
具体的な使用例として、ボタンの色がマウスがホバーされたときに変化するように設定することができます。
以下に簡単な例を示します。

具体的には クラス名blue-boxに対し大きさと色を決め、次にホバー後に一部数値（今回はwidth）を変化させるコードを二つ用意し比較し動きの違いを確認します。

（例）トランジション有りと無しの比較 一つはクラス名blue-boxに対し、transitionが無いものを、もう一つはtransitionがあるものを用意し比較してみます。
両方ともホバー後にwidthが100pxから100%に変化します。
トランジションがない場合はホバーをすると瞬時に指定された幅に変化するのに対し、トランジションの設定があるものはtransitionで指定した通り滑らかに変化します。

トランジション無し

.blue-box {
  width: 100px;
  height: 100px;
  background-color: blue;
}
.blue-box:hover {
  width: 100%;
}

トランジション有り

.blue-box {
  width: 100px;
  height: 100px;
  background-color: blue;
  transition: all 1s ease-in-out; /* transitionのプロパティに対し、時間と動きの値を入れるだけ */
}
.blue-box:hover {
  width: 100%;
}

transitionプロパティ (transition: all 1s ease-in-out;) について

ホバー後にwidthが100pxから100%に変化するtransitionプロパティに対し実際にどのような設定を行ったかというと、クラス名blue-box全体 (all) に対し、1秒かけてease-in-outという種類のイージング（動きの加減速）で動かすという設定になります。
参考サイト：ICS MEDIA「CSSイージングのお手本- ease-out, ease-in, linearの使い分け」

transitionプロパティは、下記の表のように細かく分類する事ができ、用途によって使い分けることができます。

プロパティ名読み方説明
transition-propertyトランジション・プロパティ変化が適用される CSS のプロパティを指定します。
※ 背景色や文字色などの限定的なものから要素全て (all) などの指定をおこないます。
transition-durationトランジション・デュレーション変化が始まって終わるまでの時間を指定します。
transition-timing-functionトランジション・タイミング・ファンクション変化の度合いを指定します。
※ イージング関数によって動きに変化をくわえます。
transition-delayトランジション・ディレイ変化が始まる遅延時間を指定します。
transitionトランジション上記の 4つのプロパティを一括で指定できる、ショートハンド・プロパティです。

参考サイト：株式会社アーティス「手軽なCSSアニメーション！transitionプロパティの使い方（基礎編）」

トランジション有りを分割した記法

.blue-box {
  width: 100px;
  height: 100px;
  background-color: blue;
  transition-property: all; /* アニメーションを適用するプロパティ（ここでは、変化する対象は全てとなります） */
  transition-duration: 1s; /* アニメーションの時間（秒） */
  transition-timing-function: ease-in-out; /* アニメーションのイージング関数 */
}
.blue-box:hover {
  width: 100%;
}

イージング関数の種類について

イージング関数の値(value)はTransitionプロパティとAnimationプロパティの両方で共通しています。
ホバーしたときのパラメーターの変化率は同じですが、ホバーを外したときの動きにはTransitionプロパティとAnimationプロパティで若干の違いがあります。特に、Animationプロパティの場合はその違い使用用途を考慮して記述する必要があります。

トランジションプロパティ基本のき（その2）

CSS - Transition 様々な動きについて

マウスを要素に重ねる（ホバーする）以外にも様々なアクションに応じて要素を変化させる事ができます。
主な方法としてはjavascriptを使用して、スクロールやページが読み込まれた時などをきっかけにクラス名を付与させてアニメーションさせる事ができます。
色や形、不透明度などを変化させる事ができます。

具体的には いくつか代表的な動きを、スクロールやホバーさせながら確認していきます。

スクロールアクションによるデザインエフェクト

リード文などで最も読ませたい箇所にマーカーを引くようなイメージで背景色をひき、それをスクロールによって意図したタイミングでクラス名を付与。
クラス名が付与されたタイミングで、transition: 2s; でアニメーションさせる事によって目を惹くデザインとなっております。
参考サイト：https://sakura-sup.com/jp/

See the Pen marker- line by Daizo Kaneko (@D-kaneko) on CodePen.

ホバーによるデザインエフェクト

ホバーした時にグラデーションが左から右に進み、アンダーラインが伸びるアニメーションです。
このように一つの要素内に複数のアニメーションを組み込む事も可能です。
参考サイト：https://sakura-sup.com/jp/

スクロールアクションとホバーによるデザインエフェクト

WEBサイトでよく見られるヘッダーの固定化は、グローバルナビゲーションの使い勝手を向上させるというメリットがあります。
この手法は、WEBサイトのUX向上によく使用される代表的なものです。さらに、アニメーション効果を付け加える事により、デザイン性が上がりUIの向上に繋がります。
ここでは、スクロールアクションとホバーを合わせる事で、アニメーション効果を加え、視覚的な魅力を高め、インタラクティブな体験を提供できます。
